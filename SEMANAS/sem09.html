SEMANA 9: Configuración del Entorno Java (OpenJDK) + Apache Tomcat

Fuente: Guía/diapositivas “DesarrolloBackend.pdf” (Semana 9)

Información resumida (apuntes)

Java / OpenJDK
  • Java: lenguaje base para aplicaciones backend/web empresariales.
  • JDK: kit para desarrollo (compila + ejecuta).
  • OpenJDK: implementación libre del JDK.
  • Comandos clave:
    • java --version (verifica runtime)
    • javac --version (verifica compilador)
Apache Tomcat
  • Servidor/Contenedor para aplicaciones web Java.
  • Permite trabajar con JSP y Servlets (Jakarta).
  • Escucha por defecto en puerto 8080.
  • Validación: abrir http://localhost:8080.

Checklist de verificación (antes de continuar a JSP)

Listo / OK
  • JAVA_HOME configurado correctamente.
  • PATH incluye %JAVA_HOME%\bin.
  • Comandos java y javac responden versión.
  • Tomcat abre en http://localhost:8080.
Errores comunes
  • 'java' no se reconoce → PATH mal configurado.
  • Tomcat no abre → puerto ocupado o servicio detenido.
  • Tomcat pide Java → ruta del JDK incorrecta en instalación.
  • Versiones distintas → múltiples JDK instalados sin control.

Relación entre herramientas (Java ↔ JDK ↔ IDE ↔ Tomcat)

  • Java: lenguaje y plataforma.
  • JDK/OpenJDK: herramientas para desarrollo (compilación y ejecución).
  • IDE (IntelliJ): productividad (proyectos, dependencias, ejecución, depuración).
  • Tomcat: despliegue/ejecución web (JSP/Servlets y peticiones HTTP).

  • Variables de entorno (JAVA_HOME y PATH).
  • Consola con java --version y javac --version.
  • Tomcat corriendo en navegador (localhost:8080).
  • Instalación/configuración de Tomcat mostrando selección de Java.

Reflexión

Esta semana confirmé que el backend con Java depende mucho de una buena configuración inicial. Tener OpenJDK correctamente instalado (JAVA_HOME y PATH) evita errores al compilar/ejecutar. Además, Tomcat es clave para pasar de “código Java” a “aplicación web real”, ya que actúa como contenedor para JSP/Servlets y permite probar el sistema localmente antes de avanzar a proyectos más completos.
