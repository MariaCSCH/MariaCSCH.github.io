SEMANA 12: Revisión de Laboratorio con Java y Spring Boot

Actividad enfocada en el análisis, comprensión y validación de un laboratorio desarrollado con Spring Boot

En la semana 11 se realizó la revisión integral de un laboratorio desarrollado con Java y Spring Boot. Esta actividad tuvo como objetivo comprender la estructura del proyecto, analizar el flujo de ejecución del backend y reforzar los conceptos vistos en semanas anteriores, tales como arquitectura MVC, controladores REST, configuración de dependencias y pruebas de servicios.

Objetivo de la revisión

  • Analizar la estructura de un proyecto Spring Boot
  • Comprender el rol de cada paquete y clase
  • Identificar el flujo de una petición HTTP
  • Verificar el correcto funcionamiento del laboratorio

Estructura del proyecto Spring Boot

Componente Función dentro del proyecto
Controller Recibe las solicitudes HTTP y define los endpoints del sistema
Service Contiene la lógica de negocio de la aplicación
Repository Maneja el acceso a datos y la interacción con la base de datos
Model / Entity Representa las tablas de la base de datos mediante clases Java
Application Clase principal que inicia la aplicación Spring Boot

Configuración del proyecto

Durante la revisión se analizó el archivo application.properties, donde se configuran aspectos clave del sistema como la conexión a la base de datos, el puerto del servidor y el comportamiento del framework.

Configuración Propósito
server.port Define el puerto donde se ejecuta la aplicación
spring.datasource.url URL de conexión a la base de datos
spring.jpa.hibernate.ddl-auto Controla la creación/actualización de tablas

Flujo de una petición HTTP

  1. El cliente realiza una petición HTTP (GET o POST)
  2. El Controller recibe la solicitud
  3. El Service procesa la lógica de negocio
  4. El Repository accede a la base de datos
  5. La respuesta es enviada al cliente en formato JSON

Revisión del laboratorio

  • Validación de endpoints REST
  • Revisión de anotaciones Spring
  • Comprobación de inyección de dependencias
  • Pruebas básicas con navegador o Postman

Checklist de revisión

  • El proyecto compila sin errores
  • La aplicación levanta correctamente
  • Los endpoints responden correctamente
  • La estructura MVC está bien definida

Reflexión

La revisión del laboratorio permitió reforzar los conceptos clave del desarrollo backend con Spring Boot. Analizar un proyecto funcional ayudó a comprender mejor la arquitectura, el flujo de datos y las buenas prácticas necesarias para construir aplicaciones backend robustas y escalables.
