SEMANA 14: Desarrollo Backend con Python, Flask y Jinja2

Contenido resumido y organizado del documento DesarrolloBackendconPython.pdf

En esta semana se abordó el desarrollo backend utilizando el lenguaje Python, profundizando en su integración con servidores web y frameworks modernos. Se estudió el funcionamiento del servidor Apache HTTP Server, la integración de aplicaciones Python mediante mod_wsgi, los fundamentos del lenguaje Python, las buenas prácticas de codificación (PEP-8) y el desarrollo de aplicaciones web con el framework Flask, empleando Jinja2 como motor de plantillas.

Servidor Web Apache HTTP Server

Concepto Descripción
Apache HTTP Server Servidor web de código abierto encargado de recibir solicitudes HTTP/HTTPS y entregar contenido web al cliente.
Request / Response Apache recibe una solicitud del navegador y responde con HTML, imágenes u otros recursos.
Apache Lounge Plataforma que ofrece versiones compiladas de Apache para Windows.
Configuración Se realiza mediante el archivo httpd.conf, donde se define el puerto, rutas y módulos.

Integración Python con Apache: mod_wsgi

mod_wsgi es un módulo que permite ejecutar aplicaciones web escritas en Python directamente dentro del servidor Apache, siguiendo el estándar WSGI (Web Server Gateway Interface). Gracias a esta integración, frameworks como Flask o Django pueden funcionar sin servidores intermedios adicionales.

Elemento Función
WSGI Interfaz estándar entre servidores web y aplicaciones Python
LoadModule Carga el módulo mod_wsgi dentro de Apache
WSGIScriptAlias Define el punto de entrada de la aplicación Python
<Directory> Otorga permisos de acceso a la aplicación

Lenguaje Python

Python es un lenguaje de programación interpretado, multiparadigma y de tipado dinámico. Su sintaxis clara y legible lo hace ideal para el desarrollo backend, automatización, ciencia de datos y aplicaciones web.

Característica Descripción
Indentación Define los bloques de código (no usa llaves)
Tipado dinámico No requiere declarar tipos de datos explícitamente
Multiparadigma Soporta POO, programación funcional e imperativa
Bytecode El código fuente se transforma en bytecode ejecutado por la VM de Python

Estándares de estilo PEP-8

  • Uso de 4 espacios por nivel de indentación
  • Líneas de código con máximo 79 caracteres
  • Variables y funciones en snake_case
  • Clases en PascalCase
  • Importaciones organizadas por bloques

Framework Flask

Flask es un microframework de desarrollo web en Python, diseñado para ser ligero, flexible y fácil de extender. Utiliza el patrón MVC y permite definir rutas para manejar solicitudes HTTP, renderizar vistas HTML y gestionar datos enviados por el usuario.

Concepto Función
@app.route Define las rutas de la aplicación
request Accede a los datos enviados por el cliente
response Respuesta enviada al navegador
render_template Renderiza plantillas HTML con Jinja2

Motor de Plantillas Jinja2

Jinja2 permite generar contenido HTML dinámico, separando la lógica del backend de la presentación visual. Facilita la reutilización de código y el mantenimiento de aplicaciones web.

Elemento Uso
{{ variable }} Muestra valores dinámicos
{% if %} Condicionales
{% for %} Bucles
{% extends %} Herencia de plantillas
| filtro Transformación de datos

Laboratorio (resumen breve)

  • POO en Python (clases Persona y Estudiante)
  • Aplicación Flask con formulario HTML
  • Conexión a base de datos MySQL
  • Registro de datos mediante POST

Reflexión

Esta semana permitió comprender cómo Python puede integrarse con servidores web tradicionales para construir aplicaciones backend modernas. El uso de Flask y Jinja2 facilita la creación de aplicaciones organizadas, escalables y fáciles de mantener, reforzando la separación entre lógica y presentación.
